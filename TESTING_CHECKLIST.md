# Testing Checklist

Порядок первой ручной проверки бота.

## Подготовка

1. Проверьте [`.env`](c:/Users/Павел/OneDrive/Desktop/Мои%20проекты/meal-accounting-bot/corp-meals-telegram-bot/.env):
- `BOT_TOKEN` заполнен
- `DATABASE_URL=` пустой, если используете отдельные DB-поля
- `DB_HOST/DB_PORT/DB_NAME/DB_USER/DB_PASSWORD` корректны
- `ADMIN_IDS` содержит ваш Telegram user ID

2. Выполните:

```powershell
cd .\corp-meals-telegram-bot
npm run db:init
```

3. Запустите бота:

```powershell
npm start
```

## Проверка owner

1. Откройте бота и отправьте `/start`.
2. Убедитесь, что видите inline-меню owner.
3. Откройте `Сотрудники`.
4. Добавьте 1-2 сотрудников Railship.
5. Откройте `Питание` -> `Добавить питание`.
6. Выберите сотрудника.
7. Добавьте питание на сегодня, например `250`.
8. Добавьте ещё одно питание тому же сотруднику на сегодня, например `40`.
9. Попробуйте добавить ещё `20` и убедитесь, что лимит `300 ₽` не даёт сохранить запись.
10. Откройте запись питания и проверьте:
- редактирование суммы
- редактирование даты
- редактирование комментария
- смену сотрудника
11. Откройте `Авансы` и добавьте аванс, например `15000`.
12. Откройте `Отчёты` и проверьте:
- остаток аванса
- журнал операций
- сводку по питанию за месяц

## Проверка barista

1. В owner-режиме откройте `Пользователи`.
2. Добавьте пользователя-баристу по Telegram ID.
3. Через переключение ролей откройте barista-вид.
4. Убедитесь, что:
- доступно добавление питания
- недоступны авансы, документы, реквизиты, пользователи
- можно править только свои записи

## Проверка client_viewer

1. Переключите тестовую роль на `client_viewer`.
2. Убедитесь, что:
- доступна отчётность
- доступен архив документов
- недоступно добавление питания
- недоступно редактирование авансов и реквизитов

## Проверка документов

1. В owner-режиме сформируйте акт за месяц.
2. Проверьте:
- номер акта вводится вручную
- дата акта вводится вручную
- документ открывается
3. Сформируйте акт сверки за период.
4. Проверьте:
- в таблице есть оплаты и акты
- итоговое сальдо формируется
5. Загрузите любой подписанный документ обратно в бота как файл.
6. Убедитесь, что:
- документ получает статус `signed`
- подписанный файл можно открыть из архива

## Если что-то пойдёт не так

Полезно прислать:
- какой раздел открывали;
- что нажали;
- какой текст ввели;
- что бот ответил;
- на каком шаге ожидание не совпало с результатом.
